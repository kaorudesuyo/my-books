【サーバー（GitHub / Netlify）に置くファイル構成】

  index.html            … 表示用アプリ（★トップページ）
  books.xlsx            … 本のデータ（★これを編集する）
  books.csv             … 予備データ（xlsxが読めない時だけ使われる）
  lib/
    xlsx.full.min.js    … Excel読み込みライブラリ（同梱・編集不要）
  cover/
    1.jpg 〜 100.jpg    … 書影画像（自分で用意する）

■ 重要
・ファイル名は必ず「index.html」にしてください。
  Netlifyなどのサーバーは、トップページとして index.html しか探しません。
  books.html のままだと「ページが見つかりません（404）」になります。

・上記5点は、すべて "同じ階層" に置いてください。
  フォルダごとアップロードすると1階層深くなり、404の原因になります。
  GitHubのトップ画面に index.html が直接見えている状態が正解です。

・画像のファイル名は「小文字の .jpg」で統一してください。
  サーバーは大文字小文字を区別します。1.JPG や 1.jpeg では表示されません。


【データの更新方法】

1. books.xlsx を Excel で開いて編集・保存
2. GitHub上の books.xlsx を新しいものに差し替える
3. Netlifyが自動で再公開（1〜2分）→ ブラウザをリロード

CSVへの書き出しは不要です。アプリが books.xlsx を直接読みます。

■ Excelの列（1行目のヘッダーは変更しないでください）
  id / title / author / genre / cover / amazonUrl / review

■ 注意
  ・シートは一番左のシートが読まれます
  ・cover列は「cover/1.jpg」のようにパスで記入
  ・cover列が空の本はタイトル文字のカードとして表示されます
  ・review列は後から追記していけます


【ローカルで動作確認する場合】

index.html をダブルクリックで直接開くと、ブラウザのセキュリティ制限
（file:// でのファイル読み込み禁止）によりデータを読み込めません。
必ず簡易サーバー経由で開いてください。

  フォルダ内で:  python3 -m http.server
  → ブラウザで  http://localhost:8000/


【書影画像について】

Amazonの商品ページで書影を右クリック →「画像を保存」で
cover フォルダに「番号.jpg」（例: 1.jpg）として保存してください。
